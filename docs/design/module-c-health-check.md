# 模块 C：项目递交健康度评估

> ⚠️ 本模块为后期规划，当前阶段暂不实施。

## 1. 模块概述

基于项目递交阶段 Checklist，评估项目健康度并生成改进建议。

### 1.1 定位

- 不替代现有项目管理工具
- 作为"流程质量与递交成熟度"的 AI 辅助层
- 提供客观的健康度评分和改进建议

## 2. 评估维度

### 2.1 阶段划分

| 阶段 | 说明 | 权重 |
|------|------|------|
| 设计 | 需求分析、架构设计、设计评审 | 20% |
| 开发 | 代码开发、代码评审、单元测试 | 30% |
| 测试 | 集成测试、系统测试、用户验收 | 30% |
| 递交 | 文档交付、培训、上线支持 | 20% |

### 2.2 检查项

#### 设计阶段

| 检查项 | 权重 | 数据来源 |
|--------|------|----------|
| 需求文档完整性 | 30% | Confluence |
| 设计文档评审记录 | 40% | Confluence |
| 技术方案确认 | 30% | Jira |

#### 开发阶段

| 检查项 | 权重 | 数据来源 |
|--------|------|----------|
| 代码评审覆盖率 | 40% | GitLab MR |
| 单元测试覆盖率 | 30% | CI/CD |
| 代码规范检查 | 30% | SonarQube |

#### 测试阶段

| 检查项 | 权重 | 数据来源 |
|--------|------|----------|
| 测试用例覆盖率 | 30% | TestRail |
| 缺陷收敛趋势 | 40% | Jira Bug |
| 严重缺陷关闭率 | 30% | Jira Bug |

#### 递交阶段

| 检查项 | 权重 | 数据来源 |
|--------|------|----------|
| 用户手册完成度 | 30% | Confluence |
| 培训完成度 | 30% | 培训记录 |
| 上线 Checklist | 40% | Jira |

## 3. 健康度评分

### 3.1 评分规则

```
阶段得分 = Σ(检查项得分 × 检查项权重)
总体得分 = Σ(阶段得分 × 阶段权重)
```

### 3.2 健康度等级

| 得分 | 等级 | 说明 |
|------|------|------|
| 90-100 | 🟢 优秀 | 可按计划递交 |
| 70-89 | 🟡 良好 | 需关注部分项 |
| 50-69 | 🟠 一般 | 存在风险项 |
| 0-49 | 🔴 风险 | 建议暂缓递交 |

## 5. 实施计划

### Phase 4.1：基础评估（2周）

- 实现阶段识别
- 实现 Checklist 对照
- 生成基本报告

### Phase 4.2：数据集成（2周）

- 对接 GitLab MR
- 对接 SonarQube
- 对接 Jira

### Phase 4.3：AI 建议（2周）

- 基于历史数据分析
- 生成个性化建议
- 风险预警

## 6. 数据采集

### 6.1 采集方式

| 数据源 | 采集方式 | 频率 |
|--------|----------|------|
| Jira | REST API | 每日 |
| GitLab | Webhook + API | 实时 |
| Confluence | REST API | 每日 |
| SonarQube | REST API | 每次构建 |

### 6.2 数据存储

```sql
CREATE TABLE project_health_snapshot (
    id SERIAL PRIMARY KEY,
    project_id VARCHAR(50),
    snapshot_date DATE,
    phase VARCHAR(20),
    metrics JSONB,
    overall_score DECIMAL(5,2),
    created_at TIMESTAMP DEFAULT NOW()
);
```
